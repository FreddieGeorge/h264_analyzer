## Highlights

- Added the first media/access-unit analysis foundation for video and audio.
- Added HEVC/H.265, AAC ADTS, and MP3 parser skeletons.
- Added generic audio access-unit list/property display and JSON export coverage.

## Added

- `MediaKind` and `AccessUnitKind` metadata in stream and frame analysis models.
- Container stream discovery with basic video/audio metadata.
- HEVC/H.265 parser skeleton for NAL units, VPS/SPS/PPS, VCL access units, and
  graceful unsupported slice diagnostics.
- AAC ADTS parser skeleton for 7-byte ADTS headers, audio bit fields, and
  malformed packet diagnostics.
- MP3 frame-header parser skeleton for MPEG audio frame headers, bitrate,
  sample-rate, channel-mode fields, and malformed header diagnostics.
- Mixed video/audio access-unit rows in the analysis list.
- Property-tree display for audio access units without routing them through the
  video canvas.
- JSON/CSV export paths for decoded access units, including audio analysis.

## Changed

- Project version is now `0.1.8`.
- JSON export keeps H.264 codec-specific legacy details only for H.264 frames.
- Export and list wording now uses access-unit terminology where the data may
  be video or audio.
- Documentation now describes the current audio parsing boundary and the next
  stream/packet/hex-view direction.

## Notes

- Video playback still selects the best video stream by default.
- AAC/MP3 packet analysis is opportunistic while reading container packets for
  video; there is no explicit audio stream selector or audio-only analysis mode
  yet.
- HEVC, AAC, and MP3 parser support is intentionally skeletal and focused on
  safe generic analysis output rather than full codec syntax depth.
