## Highlights

- Added a safer in-app update installation flow.
- Release builds now publish SHA256 checksum files for Windows artifacts.

## Added

- `Help -> Check for Updates` can now download the Windows installer inside
  ZStreamEye when a matching `.sha256` checksum asset is available.
- The downloaded installer is verified with SHA256 before it can be launched.
- After verification, ZStreamEye asks for confirmation, starts the installer in
  normal interactive mode, and closes the running app.
- Release workflow now uploads `.sha256` files next to the portable ZIP and
  installer.

## Changed

- The update dialog only offers the install action when both the installer and
  checksum assets are present.

## Notes

- Installation is intentionally not silent. Users still see the installer UI
  and any Windows security/UAC prompts.
- The Windows installer is not code signed yet, so Windows may show "Unknown publisher".
